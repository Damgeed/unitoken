API Documentation

OpenAI-compatible endpoint. One API key for 100+ models. Integrate in minutes.

🚀 Quick Start

Use the OpenAI Python SDK with your GlbTOKEN API key:

# pip install openai
from openai import OpenAI

client = OpenAI(
  api_key="sk-glt_...",
  base_url="https://glbtoken-api.onrender.com/api/proxy"
)

response = client.chat.completions.create(
  model="gpt-4o-mini",
  messages=[
    {"role": "user", "content": "Hello!"}
  ]
)
print(response.choices[0].message.content)

🌐 Base URL

https://glbtoken-api.onrender.com/api/proxy

This endpoint is fully compatible with the OpenAI SDK. Just change the base_url and your api_key.

🔐 Authentication

All API requests require an API key passed via the Authorization header:

Authorization: Bearer sk-glt_<your_api_key>

Generate your first API key from the Dashboard after signing up.

🤖 Available Models

List all available models with pricing:

curl https://glbtoken-api.onrender.com/api/proxy/models \
  -H "Authorization: Bearer sk-glt_..."

Or browse all models on the Models page.

💬 Chat Completions

Send a chat completion request (OpenAI-compatible):

curl https://glbtoken-api.onrender.com/api/proxy/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-glt_..." \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "What is GlbTOKEN?"}
    ],
    "temperature": 0.7,
    "max_tokens": 1000
  }'

All parameters from the OpenAI API are supported: stream, frequency_penalty, top_p, etc.

⚡ Streaming

Server-Sent Events (SSE) streaming is fully supported:

from openai import OpenAI

client = OpenAI(
  api_key="sk-glt_...",
  base_url="https://glbtoken-api.onrender.com/api/proxy"
)

stream = client.chat.completions.create(
  model="gpt-4o-mini",
  messages=[{"role": "user", "content": "Tell me a story"}],
  stream=True
)
for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")

🟩 Node.js Example

import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "sk-glt_...",
  baseURL: "https://glbtoken-api.onrender.com/api/proxy"
});

const completion = await client.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "Hello!" }]
});
console.log(completion.choices[0].message.content);

⚠️ Error Codes

Code Meaning
401 Invalid or missing API key
402 Insufficient token balance
404 Model not found or not available
429 Rate limited — slow down requests

✅ Best Practices

  • Rotate keys regularly — Generate new API keys from your Dashboard and revoke old ones.
  • Use separate keys per project — Helps track usage and limit blast radius.
  • Set max tokens — Always set max_tokens to control costs.
  • Monitor usage — Check your Dashboard for real-time token consumption.
  • Handle 402 errors — Top up before your balance runs out to avoid service interruption.
